# 개발자 웹 이력서 프로젝트 로드맵

## 📋 프로젝트 개요
HTML, CSS, JavaScript, Tailwind CSS를 활용하여 반응형 웹 개발자 이력서 사이트 구축

---

## 🎯 프로젝트 구조

```
portfolio/
├── index.html           # 메인 페이지
├── css/
│   └── styles.css       # 커스텀 스타일
├── js/
│   └── script.js        # 인터랙션 로직
├── assets/
│   ├── images/          # 프로필 사진 등
│   └── icons/           # 아이콘 에셋
└── README.md            # 프로젝트 설명
```

---

## 🚀 개발 단계

### Phase 1: 기본 구조 설정 (Day 1)
- [ ] 프로젝트 폴더 구조 생성
- [ ] HTML 기본 레이아웃 작성
  - 헤더/네비게이션
  - 소개 섹션
  - 경력 섹션
  - 기술 스택 섹션
  - 프로젝트 섹션
  - 연락처 섹션
  - 푸터
- [ ] Tailwind CSS 설정 (CDN 또는 로컬 설정)

### Phase 2: 스타일링 (Day 2)
- [ ] Tailwind CSS로 기본 스타일 적용
  - 색상 스킴 정의 (다크/라이트 모드 고려)
  - 타이포그래피 설정
  - 레이아웃 디자인
- [ ] 반응형 디자인 구현
  - 모바일 (sm: < 640px)
  - 태블릿 (md: 640px ~ 1024px)
  - 데스크톱 (lg: > 1024px)
- [ ] 커스텀 CSS 추가 (필요시)

### Phase 3: 인터랙션 구현 (Day 3)
- [ ] JavaScript 기능 추가
  - 스크롤 네비게이션 활성화
  - 부드러운 스크롤
  - 모바일 메뉴 토글
  - 다크/라이트 모드 전환
- [ ] 애니메이션 효과
  - 페이드 인/아웃
  - 슬라이드 인/아웃
  - 호버 이펙트

### Phase 4: 콘텐츠 입력 (Day 4)
- [ ] 개인 정보 작성
  - 이름, 직책, 짧은 소개
  - 프로필 사진 등록
- [ ] 경력 정보 작성
  - 근무 경력 (회사, 직책, 기간, 설명)
- [ ] 기술 스택 작성
  - 프론트엔드: HTML, CSS, JavaScript, React 등
  - 백엔드: Node.js, Python 등
  - 도구: Git, Docker, 등
- [ ] 주요 프로젝트 작성
  - 프로젝트 제목, 설명, 기술 스택, 링크

### Phase 5: 최적화 및 배포 (Day 5)
- [ ] 성능 최적화
  - 이미지 최적화
  - CSS/JS 미니피케이션
  - 로딩 속도 개선
- [ ] SEO 최적화
  - Meta 태그 설정
  - Open Graph 태그 추가
  - 시맨틱 HTML 적용
- [ ] 브라우저 호환성 테스트
- [ ] 배포
  - GitHub Pages / Netlify / Vercel 등 선택
  - 도메인 설정 (선택사항)

## 📊 이력서 주요 섹션

### 1️⃣ 헤더 (Header)
```
- 이름 / 직책
- 한 줄 소개
- 프로필 사진
- 네비게이션 메뉴
```

### 2️⃣ 소개 (About)
```
- 자기소개 (2-3문장)
- 핵심 강점 3가지
- CTA 버튼 (이력서 다운로드, 연락하기)
```

### 3️⃣ 경력 (Experience)
```
- 회사명, 직책, 기간
- 주요 역할 및 성과
- 사용 기술
```

### 4️⃣ 기술 스택 (Skills)
```
- 프로그래밍 언어
- 웹 프레임워크/라이브러리
- 개발 도구 및 플랫폼
- 기타 기술
```

### 5️⃣ 프로젝트 (Projects)
```
- 프로젝트 제목
- 간단한 설명
- 사용 기술 (태그)
- 프로젝트 이미지
- 깃허브/라이브 링크
```

### 6️⃣ 연락처 (Contact)
```
- 이메일
- 깃허브 링크
- 링크드인 링크
- 기타 SNS 링크
- 연락처 폼 (선택사항)
```

## 💡 개발 팁

1. **Tailwind CSS 활용**: 유틸리티 클래스를 최대한 활용하여 빠른 개발
2. **반응형 디자인**: 모바일 우선 접근법 사용
3. **접근성**: 시맨틱 HTML, 키보드 네비게이션 지원
4. **성능**: 이미지 최적화, 불필요한 CSS 제거
5. **사용자 경험**: 명확한 CTA, 빠른 로딩 속도
